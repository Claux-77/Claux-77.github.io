Input System的基本使用

1. Input System实现了一个可使用任何类型的输入设备来控制unity内容的系统。旨在替换unity旧的输入管理器(UnityEngine.Input)

2. Action Maps：动作表，可以依据不同的情况使用不同的控制方式。比如WASD在UI界面和游戏界面拥有不同的响应行为，那么就使用不同的动作表

3. 安装和使用。

  • Package Manager中搜索Input System进行安装。需重启unity
  • Project窗口右键新建Input Actions，双击打开即可配置
  • 可新建一个动作表，其中配置Actions(如Move等)，绑定其对应的操作行为
    • Action Type:动作类型。
    • Control Type:Action Type选择Value时才会出现，即控制类型。
  • Move右侧加号可新加绑定。键盘绑定使用2d vector，手柄直接绑定。配置完成手柄和键盘：
  • 可再添加方向键的绑定。记得Save Asset
  • 点击配置文件Input Actions，可选择自动生成对应的C#文件
  • 新建C#文件并继承生成的代码文件中的IGameplayActions接口，Coding：
[CreateAssetMenu(menuName = "Player Input")]
public class PlayerInput : ScriptableObject,PlayerInputAction.IGameplayActions
{
    private PlayerInputAction playerInputAction;
    public event UnityAction<Vector2> onMove; // 开始移动事件
    public event UnityAction onStopMove; // 停止移动事件
    private void OnEnable()
    {
        playerInputAction = new PlayerInputAction();
        //登记playerInputAction的回调函数为自己。
        playerInputAction.Gameplay.SetCallbacks(this);
    }
    private void OnDisable()
    {
        DisableAllInputs();
    }
    /// <summary>
    /// 启用此动作表。
    /// </summary>
    public void EnableGameplayInput()
    {
        playerInputAction.Gameplay.Enable();

        //隐藏以及锁定鼠标光标
        Cursor.visible = false;
        Cursor.lockState = CursorLockMode.Locked;
    }
    /// <summary>
    /// 禁用此动作表。
    /// </summary>
    public void DisableAllInputs()
    {
        playerInputAction.Gameplay.Disable();
    }
    public void OnMove(InputAction.CallbackContext context)
    {
        //玩家按住按键时触发事件并传入一个Vector2的参数
        if (context.performed)
        {
            onMove?.Invoke(context.ReadValue<Vector2>());
        }
        //松开按键时触发此事件
        if (context.canceled)
        {
            onStopMove?.Invoke();
        }
    }
}
